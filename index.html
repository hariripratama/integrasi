1

Pendahuluan

Modul praktikum ini menguraikan perancangan sistem backend RESTful API Manajemen Data Panen & Profil Petani berbasis runtime environment Node.js. Implementasi ini berfungsi sebagai pembuktian praktis dari implementasi sepuluh pilar arsitektur RESTful global seperti yang diilustrasikan pada acuan skema rancangan ekosistem perkuliahan.

Prinsip RESTful API Penerapan Spesifik pada Project Panen & Petani
Penamaan Resource Mengisolasi objek data menggunakan nomina jamak berbeda sesuai konteks:
/api/v1/panen dan /api/v1/petani
Operasi HTTP (CRUD) GET POST PUT PATCH DELETE
Control Versioning Menghindari tabrakan rute struktural dengan menyisipkan URL prefix path: /api/v1/
Content Negotiation Proses tukar-menukar data payload klien dan server seutuhnya dikunci dalam format standar JSON
Penyaringan Query Membatasi data dinamis menggunakan pencarian kata kunci filter parameter URL:
?jenis_tanaman=... atau ?kelompok_tani=... atau ?id=...
Status Code Standar Representasi indikator status terklasifikasi: 200 (OK), 201 (Created), 401 (Unauthorized), 404 (Not Found)
Keamanan Gerbang Validasi hak akses masuk terproteksi wajib melampirkan komponen header kustom: X-API-Key
Sifat Idempotent Method GET, PUT, dan DELETE didesain konsisten agar tidak merusak data jika dipanggil berulang kali.
HATEOAS Navigasi Menyematkan struktur hypermedia tautan referensi otomatis di dalam properti objek respons berkode _links
Prinsip Stateless Server tidak menyimpan histori memori session; setiap request diproses secara independen dan utuh.

Persiapan Environment

A. Mengaktifkan Local Server

Inisialisasi dependensi library node_modules dan eksekusi development server melalui perintah terminal berikut:

# Menjalankan skrip otomatis nodemon development

npm run dev

Setelah perintah di atas dieksekusi, terminal akan melakukan kompilasi berkas utama. Pastikan log terminal menampilkan status sukses berjalan aktif seperti pada screenshot di bawah ini:

Gambar: Tampilan log sirkulasi kompilasi backend server berjalan aktif pada port 3000.

B. Alternatif Pengujian Eksternal: Thunder Client & Postman

Pengujian dapat memanfaatkan ekstensi **Thunder Client** yang terintegrasi langsung di dalam lingkungan kerja VS Code atau aplikasi **Postman**. Kedua alat ini berfungsi sebagai klien untuk menembak endpoint HTTP lokal dengan menyertakan konfigurasi header rahasia.

Struktur Proyek Terstruktur (src)

Berikut adalah representasi susunan skema berkas fungsional baru yang memisahkan antara modul data dan kontroler rute **Panen** serta **Petani**:

PANEN PETANI/ ├── node_modules/ ├── src/ │ ├── data/ │ │ ├── panen.js // Model data / local state mock database panen │ │ └── petani.js // Model data / local state mock database petani (BARU) │ ├── middleware/ │ │ ├── apiKey.js // Interseptor validasi X-API-Key │ │ └── logger.js // Tracker log aktivitas request masuk │ ├── routes/ │ │ ├── panen.js // Manajemen endpoint modular rute /api/v1/panen │ │ └── petani.js // Manajemen endpoint modular rute /api/v1/petani (BARU) │ └── server.js // Berkas utama konfigurasi & inisialisasi Express app ├── package.json
4

Penjelasan Server Entry Point (server.js)

File gerbang utama `server.js` meregistrasikan rute modular secara terpisah untuk menjaga kode tetap clean dan modular:

const express = require('express');

const app = express();

app.use(express.json()); // Built-in middleware untuk parsing JSON request payload body

// Menghubungkan rute modular terproteksi masing-masing resource

app.use('/api/v1/panen', apiKeyAuth, require('./routes/panen'));

app.use('/api/v1/petani', apiKeyAuth, require('./routes/petani')); // Integrasi Rute Petani

Uji Proteksi Gerbang Keamanan (Tanpa API Key)

401 Unauthorized

  "success": false,
  "error": { "code": 401, "message": "API Key diperlukan. Kirim header: X-API-Key" }
}

Kesimpulan

Berdasarkan seluruh rangkaian perancangan, implementasi, hingga tahapan pengujian komprehensif yang telah dilakukan menggunakan ekstensi Thunder Client, Postman, dan cURL, diperoleh beberapa kesimpulan penting sebagai berikut:

  • Arsitektur Multi-Resource RESTful yang Konsisten: Modul backend sukses mengadopsi pilar utama RESTful API dengan mengisolasi rute entitas jamak yang jelas, yakni `/api/v1/panen` untuk log hasil pertanian, dan `/api/v1/petani` untuk basis profil kelompok tani desa.
  • Keamanan Data Terjamin: Pemasangan gerbang interceptor middleware autentikasi kustom berhasil memvalidasi parameter kunci `X-API-Key` secara global pada kedua modul resource data tersebut.
  • Independensi & Kesiapan Integrasi: Hasil pengujian eksternal via terminal CLI cURL menegaskan keandalan backend *stateless* ini yang modular dan siap diintegrasikan dengan platform eksternal frontend maupun aplikasi IoT di masa depan.
